【L3模组设计】
  世界规则:
    [WR0] 创作者豁免
      规则: 所有世界规则只约束KP和玩家，模组创作者不受世界规则约束
      范围: ['meta']
      性质: True
    [WR1] 不可阻挡的吞噬
      规则: 列车末端存在一只无法理解、无法对抗的巨口，正缓慢而不可逆地吞噬车厢。一旦车厢被吞噬，其所在空间彻底消失，无法返回。
      范围: 整个模组空间，特别是后方车厢。巨口吞噬是所有时间压力和强制前进的根源。
      性质: 绝对不可违反。没有任何方法可以阻止、摧毁或改变巨口的吞噬进程。
    [WR2] 线性强制的单向旅行
      规则: 列车只能向前方车厢前进。回头探索虽然初期可能，但由于巨口的追赶和后方空间的消失，回头会变得越来越危险，最终完全不可能。门可能突然关死，后方变成虚无。
      范围: 所有车厢的连接处。这是一个物理法则般的限制，确保玩家被推向未知的前方。
      性质: 在一般情况下严格存在。初期可有限度地返回，但随着时间推移逐渐绝对化。
    [WR3] 梦境与现实的叠加
      规则: 调查员所处的电车空间是现实与梦境的交界，或为一场集体幻觉。报纸上的日期混乱、结局醒来方式的不同，都暗示这里的时间、空间和物理逻辑与现实不同。
      范围: 整个模组世界。所有超自然现象、怪物存在和最终逃脱方式都受此规则影响。
      性质: 绝对。这个空间并不是真实的物理电车，其中的法则混合了噩梦的逻辑，即使使用现实手段也无法用常规逻辑破解。
    [WR4] 循声者的听觉猎杀
      规则: 怪物“循声者”没有视觉，完全依赖声音定位。任何足以为其提供方位的声响都会立即引发攻击。它们对非自然响动极其敏感，且力量巨大，几乎不可正面战胜。
      范围: 主要适用于2号车厢，但任何可能制造声响惊动它们的行为都受此规则制约。
      性质: 在2号车厢内绝对有效。怪物永远不会被光线影响，也不会主动免疫声音。正面战斗几乎必然导致死亡，因此可以被视为一个绝对无法战胜的规则。
    [WR5] 预知与循环的时间错位
      规则: 报纸上出现了“第二天”的新闻，描述的是正在发生的事件，暗示时间并非线性，或者调查员陷入了一个注定发生悲剧的循环。这创造了宿命感和对失败的恐惧。
      范围: 线索报纸所预示的内容，以及结局醒来的差异。这条规则定义了模组的叙事结构：可能是一次不断重复的噩梦，或者是对未来的窥视。
      性质: 作为叙事规则存在，调查员无法改变报纸上的报道内容，但可以通过自己的选择改变“这一次”的结果。
  核心驱动力: 一列末班电车被拖入了一个由集体恐惧或某个不可名状存在编织的噩梦空间。电车的后端被一只象征性的巨大嘴巴缓慢吞噬，代表未解决的恐惧或终将降临的死亡不断逼近。所有事件都是这股驱动力在逼迫调查员做出反应：是向前冲破幻境，还是向后被恐惧吞没。一切的底层逻辑是：只有直面恐惧并加速穿越它，才能醒来。
  叙事线:
    [主线] 常暗之箱的逃脱
      大纲: 起：调查员在空荡的6号车厢醒来，发现列车异常，获得便签和地图暗示后方危险。承：探索后车厢目睹巨口，在5号车厢发现预示性报纸，时间吞噬加速，迫使向前。转：在4号车厢救援乘务员，获取关键情报和钥匙下落；在3号车厢找回钥匙和个人物品；于2号车厢冒险穿越循声者猎场。合：抵达驾驶室，面临最终加速或减速的核心抉择，根据选择达成不同结局。关键转折点：拯救乘务员成功与否影响情报获取；在2号车厢的潜行成败决定是否能到达驾驶室；驾驶室中的选择直接决定三种结局。
      关键场景: 6号车厢, 7号车厢, 5号车厢, 4号车厢, 3号车厢, 2号车厢, 先头车厢
  基调约束:
  genre:
    克苏鲁恐怖（梦境/异空间逃生）
  forbidden:
    现代武器有效对抗怪物（如枪支、炸药）, 正面击败循声者或巨口, 无视声音可以安全通过2号车厢, 从后方车门逃出电车, 轻松的、毫无代价的逃脱
  recommended:
    逐渐增强的、无法摆脱的紧迫感, 理智值不断受到打击，即使成功也可能留下后遗症, 道德困境：信任与猜疑、拯救与抛弃, 对未知的恐惧胜过直接呈现的怪物, 微弱但真实存在的希望之光（如加速逃脱）, 梦境逻辑的不确定性，让玩家怀疑一切
  narrative_style:
    压抑、封闭的箱庭式恐怖。注重环境氛围和心理描写，让恐惧从日常场景（末班电车）中滋生。时间压力和声音机制是紧张感的主要来源。避免廉价的跳跃惊吓，而是通过逐渐揭示的恐怖事实和不断恶化的处境来压迫玩家。
  当前场景设计意图:
  purpose:
    作为起始场景，营造诡异的孤独氛围，让调查员意识到列车异常，并通过便签和地图提供最初的两种冲突信息：必须前进，以及后方有未知危险被隐藏。同时让玩家进入探索状态。
  key_threat:
    未知的漫长旅途，后方被涂改的地图暗示危险。
  notes:
    一开始就给出明确的动机：前进，没有退路。

Entity 字段规则：
- id: 全局唯一，patch 用 SI1/SI2...，auto_trigger 用 SAT1/SAT2...，event 用 SE1/SE2...
- entity_type: interaction / auto_trigger / event
- scene: 所在场景名（中文）
- name: 简短动作名
- type: 关联技能名（如"侦查""急救"），不涉及检定填"无"
- requirement: 硬性前置条件用 entity ID + AND/OR/() 表达复合关系（如 SI1 AND SI2、(SI1 OR SI2) AND SI3），裸 entity ID 默认指该实体成功完成。无条件填空字符串。需要特殊条件（如实体检定失败、调查员理智极度崩溃等）在 "||" 后用自然语言描述。requirement 可描述是否需要消耗常见物品及数量（如"需要消耗1个急救包"）
- trigger: 触发场景——描述什么情况下玩家可以执行此互动。不要和 requirement 混淆
- result: 直接结果——互动直接产生的可感知结果。如果会触发游戏结局，必须以 ##END_结局名称:结局简述## 开头。涉及技能检定时 result 填 "##GRADED##"（占位标记），side_effects 留空，所有结果文字写入 graded_result
- side_effects: 间接后果——与 result 不重合的附带影响。自然语言字符串列表。无条件则为空列表
- difficulty: None / regular / hard / extreme；不涉及检定则为 None
- graded_result: type 不为"无"时填写。四等级：on_failure=检定失败、on_regular=常规成功、on_hard=困难成功（≤技能值/2）、on_extreme=极难成功（≤技能值/5）。若原文未区分等级，各等级可描述相同内容
- entities 的 result/side_effects 不涉及进入与怪物的战斗/对抗/追捕（怪物遭遇和战斗由 game loop 运行时统一管理）。可以声明怪物出现，但不描述进入和怪物的对砍/战斗
- @标记可嵌入 result / side_effects / graded_result 任意字段中，与普通文本混合。间接/附带影响使用 @函数(参数) 语法：@spawn_enemy(enemy_ref="名称", scene="场景", quantity=1) / @grant_weapon(weapon_ref="名称", scene="场景", quantity=1) / @stat_change(stat_name="属性", delta=-1) / @item_gain(item_name="物品", quantity=1) / @consume_item(item_name="物品", quantity=1) / @npc_state_change(npc_name="名称", new_state="状态") / @npc_follow(npc_name="名称", follow=true). @grant_weapon 若 scene 为空，表示直接授予调查员（等价于搜索拾取武器的流程，只是触发条件不同）

创作规则：
- 只添加必要的entity，不要过度扩充
- structural 仅在玩家行为确实需要时才使用
- justification 必须引用L3设计意图
- 直接输出 JSON

Entity 字段规则：
- id: 全局唯一，patch 用 SI1/SI2...，auto_trigger 用 SAT1/SAT2...，event 用 SE1/SE2...
- entity_type: interaction / auto_trigger / event
- scene: 所在场景名（中文）
- name: 简短动作名
- type: 关联技能名（如"侦查""急救"），不涉及检定填"无"
- requirement: 硬性前置条件用 entity ID + AND/OR/() 表达复合关系（如 SI1 AND SI2、(SI1 OR SI2) AND SI3），裸 entity ID 默认指该实体成功完成。无条件填空字符串。需要特殊条件（如实体检定失败、调查员理智极度崩溃等）在 "||" 后用自然语言描述。requirement 可描述是否需要消耗常见物品及数量（如"需要消耗1个急救包"）
- trigger: 触发场景——描述什么情况下玩家可以执行此互动。不要和 requirement 混淆
- result: 直接结果——互动直接产生的可感知结果。如果会触发游戏结局，必须以 ##END_结局名称:结局简述## 开头。涉及技能检定时 result 填 "##GRADED##"（占位标记），side_effects 留空，所有结果文字写入 graded_result
- side_effects: 间接后果——与 result 不重合的附带影响。自然语言字符串列表。无条件则为空列表
- difficulty: None / regular / hard / extreme；不涉及检定则为 None
- graded_result: type 不为"无"时填写。四等级：on_failure=检定失败、on_regular=常规成功、on_hard=困难成功（≤技能值/2）、on_extreme=极难成功（≤技能值/5）。若原文未区分等级，各等级可描述相同内容

Entity 字段规则：
- id: 全局唯一，patch 用 SI1/SI2...，auto_trigger 用 SAT1/SAT2...，event 用 SE1/SE2...
- entity_type: interaction / auto_trigger / event
- scene: 所在场景名（中文）
- name: 简短动作名
- type: 关联技能名（如"侦查""急救"），不涉及检定填"无"
- requirement: 硬性前置条件用 entity ID + AND/OR/() 表达复合关系（如 SI1 AND SI2、(SI1 OR SI2) AND SI3），裸 entity ID 默认指该实体成功完成。无条件填空字符串。需要特殊条件（如实体检定失败、调查员理智极度崩溃等）在 "||" 后用自然语言描述。requirement 可描述是否需要消耗常见物品及数量（如"需要消耗1个急救包"）
- trigger: 触发场景——描述什么情况下玩家可以执行此互动。不要和 requirement 混淆
- result: 直接结果——互动直接产生的可感知结果。如果会触发游戏结局，必须以 ##END_结局名称:结局简述## 开头。涉及技能检定时 result 填 "##GRADED##"（占位标记），side_effects 留空，所有结果文字写入 graded_result
- side_effects: 间接后果——与 result 不重合的附带影响。自然语言字符串列表。无条件则为空列表
- difficulty: None / regular / hard / extreme；不涉及检定则为 None
- graded_result: type 不为"无"时填写。四等级：on_failure=检定失败、on_regular=常规成功、on_hard=困难成功（≤技能值/2）、on_extreme=极难成功（≤技能值/5）。若原文未区分等级，各等级可描述相同内容
- entities 的 result/side_effects 不涉及进入与怪物的战斗/对抗/追捕（怪物遭遇和战斗由 game loop 运行时统一管理）。可以声明怪物出现，但不描述进入和怪物的对砍/战斗
- @标记可嵌入 result / side_effects / graded_result 任意字段中，与普通文本混合。间接/附带影响使用 @函数(参数) 语法：@spawn_enemy(enemy_ref="名称", scene="场景", quantity=1) / @grant_weapon(weapon_ref="名称", scene="场景", quantity=1) / @stat_change(stat_name="属性", delta=-1) / @item_gain(item_name="物品", quantity=1) / @consume_item(item_name="物品", quantity=1) / @npc_state_change(npc_name="名称", new_state="状态") / @npc_follow(npc_name="名称", follow=true). @grant_weapon 若 scene 为空，表示直接授予调查员（等价于搜索拾取武器的流程，只是触发条件不同）

- structural 仅在玩家行为确实需要时才使用
- justification 必须引用L3设计意图

Entity 字段规则：
- id: 全局唯一，patch 用 SI1/SI2...，auto_trigger 用 SAT1/SAT2...，event 用 SE1/SE2...
- entity_type: interaction / auto_trigger / event
- scene: 所在场景名（中文）
- name: 简短动作名
- type: 关联技能名（如"侦查""急救"），不涉及检定填"无"
- requirement: 硬性前置条件用 entity ID + AND/OR/() 表达复合关系（如 SI1 AND SI2、(SI1 OR SI2) AND SI3），裸 entity ID 默认指该实体成功完成。无条件填空字符串。需要特殊条件（如实体检定失败、调查员理智极度崩溃等）在 "||" 后用自然语言描述。requirement 可描述是否需要消耗常见物品及数量（如"需要消耗1个急救包"）
- trigger: 触发场景——描述什么情况下玩家可以执行此互动。不要和 requirement 混淆
- result: 直接结果——互动直接产生的可感知结果。如果会触发游戏结局，必须以 ##END_结局名称:结局简述## 开头。涉及技能检定时 result 填 "##GRADED##"（占位标记），side_effects 留空，所有结果文字写入 graded_result
- side_effects: 间接后果——与 result 不重合的附带影响。自然语言字符串列表。无条件则为空列表
- difficulty: None / regular / hard / extreme；不涉及检定则为 None
- graded_result: type 不为"无"时填写。四等级：on_failure=检定失败、on_regular=常规成功、on_hard=困难成功（≤技能值/2）、on_extreme=极难成功（≤技能值/5）。若原文未区分等级，各等级可描述相同内容
- entities 的 result/side_effects 不涉及进入与怪物的战斗/对抗/追捕（怪物遭遇和战斗由 game loop 运行时统一管理）。可以声明怪物出现，但不描述进入和怪物的对砍/战斗
- @标记可嵌入 result / side_effects / graded_result 任意字段中，与普通文本混合。间接/附带影响使用 @函数(参数) 语法：@spawn_enemy(enemy_ref="名称", scene="场景", quantity=1) / @grant_weapon(weapon_ref="名称", scene="场景", quantity=1) / @stat_change(stat_name="属性", delta=-1) / @item_gain(item_name="物品", quantity=1) / @consume_item(item_name="物品", quantity=1) / @npc_state_change(npc_name="名称", new_state="状态") / @npc_follow(npc_name="名称", follow=true). @grant_weapon 若 scene 为空，表示直接授予调查员（等价于搜索拾取武器的流程，只是触发条件不同）

Entity 字段规则：
- id: 全局唯一，patch 用 SI1/SI2...，auto_trigger 用 SAT1/SAT2...，event 用 SE1/SE2...
- entity_type: interaction / auto_trigger / event
- scene: 所在场景名（中文）
- name: 简短动作名
- type: 关联技能名（如"侦查""急救"），不涉及检定填"无"
- requirement: 硬性前置条件用 entity ID + AND/OR/() 表达复合关系（如 SI1 AND SI2、(SI1 OR SI2) AND SI3），裸 entity ID 默认指该实体成功完成。无条件填空字符串。需要特殊条件（如实体检定失败、调查员理智极度崩溃等）在 "||" 后用自然语言描述。requirement 可描述是否需要消耗常见物品及数量（如"需要消耗1个急救包"）
- trigger: 触发场景——描述什么情况下玩家可以执行此互动。不要和 requirement 混淆
- result: 直接结果——互动直接产生的可感知结果。如果会触发游戏结局，必须以 ##END_结局名称:结局简述## 开头。涉及技能检定时 result 填 "##GRADED##"（占位标记），side_effects 留空，所有结果文字写入 graded_result
- side_effects: 间接后果——与 result 不重合的附带影响。自然语言字符串列表。无条件则为空列表
- difficulty: None / regular / hard / extreme；不涉及检定则为 None
- graded_result: type 不为"无"时填写。四等级：on_failure=检定失败、on_regular=常规成功、on_hard=困难成功（≤技能值/2）、on_extreme=极难成功（≤技能值/5）。若原文未区分等级，各等级可描述相同内容
- entities 的 result/side_effects 不涉及进入与怪物的战斗/对抗/追捕（怪物遭遇和战斗由 game loop 运行时统一管理）。可以声明怪物出现，但不描述进入和怪物的对砍/战斗
- @标记可嵌入 result / side_effects / graded_result 任意字段中，与普通文本混合。间接/附带影响使用 @函数(参数) 语法：@spawn_enemy(enemy_ref="名称", scene="场景", quantity=1) / @grant_weapon(weapon_ref="名称", scene="场景", quantity=1) / @stat_change(stat_name="属性", delta=-1) / @item_gain(item_name="物品", quantity=1) / @consume_item(item_name="物品", quantity=1) / @npc_state_change(npc_name="名称", new_state="状态") / @npc_follow(npc_name="名称", follow=true). @grant_weapon 若 scene 为空，表示直接授予调查员（等价于搜索拾取武器的流程，只是触发条件不同）
